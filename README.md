# 리눅스 프로세스 관리 명령어 정리
---

## 
이 문서에서는 리눅스 명령어 top, ps, jobs, kill 네가지 명령어를 설명합니다
---

## 1. 'top' 
### 실시간 프로세스 모니터
- GPU 사용량, 메모리 사용량, 실행중인 프로세스 등을 실시간으로 갱신하면서 보여주는 명령어 입니다.
- windows의 작업관리자와 같은 역할입니다.

### 주요기능
- 현재 실행 중인 프로세스 목록 확인
- CPU·메모리 사용량 확인
- 부하가 큰 프로세스 추적

### 자주 사용하는 조작
- `k` : 프로세스 종료
- `r` : 우선순위 변경
- `q` : 종료

## 2. 'ps' 
### 실행중인 프로세스 정보 확인
- `top`처럼 실시간이 아니라, 명령을 입력한 그 순간의 프로세스 목록만 표시합니다.

### 자주 사용하는 조작
| 명령 | 설명 |
|----|----|
| `ps` | 현재 셸의 프로세스만 출력 |
| `ps -e` 또는 `ps -A` | 시스템의 모든 프로세스 표시 |
| `ps -f` | 풀 포맷(full format) 표시 |
| `ps aux` | 전체 프로세스 자세히 표시 |

## 3. 'jobs' 
### 현재 쉘에서 실행 중인 백그라운드 작업 목록 확인
- 한 터미널에서 실행한 백그라운드 작업만 관리합니다.
- 시스템 전체 프로세스는 보지 못함니다.

### 자주 사용하는 조작
| 명령 | 설명 |
|----|----|
| `job` | 현재 쉘의 background 작업 목록 출력 |
| `command &`| 프로세스를 백그라운드에서 실행 |
| `fg %번호` | 작업을 foreground로 가져오기 |
| `bg %번호` | 작업을 백그라운드로 전환 |

## 4. 'kill' 

### 프로세스에 신호를 보내는 명령
- 프로세스를 종료하거나 특정 동작을 요청하는 신호를 보내는 명령어입니다


### 자주 사용하는 신호
| 신호 | 번호| 설명|
|----|----|----|
| `SIGTERM` | 15 | 정상 종료 요청|
| `SIGKILL`| 9 | 강제종료 |
| `SIGKILL` | 1 | 설정변경 / 재시작 |

---
## 전체요약
| 명령어 | 특징| 목적|
|----|----|----|
| `top` | 실시간 모니터링 | CPU, 메모리 등 실시간 확인|
| `ps`| 프로세스 스냅샷 출력 | 현재 프로세스 정보 조회 |
| `jobs` | 백그라운드 작업 관리 | 셸 내 작업 이동/관리 |
| `kill` | 프로세스 제어 | 종료/재시작 등 signal 전달 |
